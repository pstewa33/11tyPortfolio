Overview

Built and maintained an internal component library and page templates based on the United States Web Design System (USWDS). The library was used to standardize UI across multiple public sector website redesign projects, improve accessibility compliance, and enable faster, scalable page creation.

My Role

  • Implemented all USWDS components in the internal system using HTML, CSS, and Markdown

  • Developed custom web components for reusable UI patterns

  • Created template-based page layouts for common internal pages

  • Managed version control and deployment using GitHub and Netlify

  • Transitioned from Jekyll to Eleventy for improved build performance and flexibility

Tech Stack

  • HTML / CSS / Markdown

  • Custom Web Components

  • Static Site Generators: Jekyll → Eleventy

  • Deployment: Netlify

  • Version Control: GitHub

Key Engineering Challenges and Solutions

  1. Component Architecture

  • Designed a modular component structure to maximize reuse

  • Standardized CSS classes and utility patterns for consistency

  • Built components to support multiple variants (e.g., buttons, cards, navigation)

  1. Template System

  • Created reusable page layouts using static site generation

  • Enabled fast creation of new pages without duplicating markup

  • Structured templates to support scalability as new internal sites were added

  1. Accessibility

  • Ensured semantic HTML across all components

  • Applied ARIA roles where needed for dynamic components

  • Focused on keyboard navigation and color contrast compliance

  1. Build & Deployment Optimization

  • Migrated from Jekyll to Eleventy to simplify build process and improve flexibility

  • Deployed via Netlify with automated builds from GitHub

  • Implemented version control workflow to manage component updates safely

Impact

  • Reduced duplication and inconsistencies across internal pages

  • Enabled faster creation of new pages and components

  • Established a reusable, maintainable component library that can scale with the organization

  • Improved accessibility compliance and overall user experience consistency
