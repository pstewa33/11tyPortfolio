Overview

Built a functional, code-based mini-site prototype using internal design system components, brand styling, and headless CMS integration. Delivered an interactive, high-fidelity prototype to the client.

My Role

  • Wrote HTML/CSS/JS prototypes using components from the internal USWDS-based library

  • Applied brand-specific styling to create high-fidelity interactive frames

  • Connected Jekyll to Contentful for dynamic content management

  • Deployed the prototype on Netlify

  • Delivered a functional mini-site demonstrating accurate interactions, branding, and content

Tech Stack

  • HTML / CSS / Markdown

  • Custom Web Components

  • Jekyll + Contentful integration

  • Deployment: Netlify

  • Version Control: GitHub

Key Engineering Challenges and Solutions

  1. Component Reuse – Leveraged internal library components to maintain consistency and speed up development.

  2. High-Fidelity Prototyping – Applied brand styling and interactions in code rather than static mockups.

  3. CMS Integration – Connected Jekyll to Contentful for dynamic content delivery.

  4. Deployment & Delivery – Hosted on Netlify for client-accessible functional prototype.

Impact

  • Delivered a fully interactive prototype instead of static mockups

  • Demonstrated scalability and reusability of internal design system

  • Enabled client to preview interactions, branding, and content in a realistic environment
